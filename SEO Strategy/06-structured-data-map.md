**Governing rules (apply to every row below):**

1. **Every structured-data field must match visible page content.** Google structured-data guidelines: markup must describe content users can see on the page. No hidden, inflated, or aspirational values.
2. **Validate every deployment three ways:** Rich Results Test (`https://search.google.com/test/rich-results`), Schema.org validator (`https://validator.schema.org/`), and GSC enhancement reports (after GSC verification — see `09-analytics-baseline.md`).
3. **Never mark up fake data.** The live artist/studio inventory, ratings (4.7–4.9), and review counts (65–154) are seed/demo data from `app/src/data/mockData.ts` served verbatim in production (R1 F5: `GET /api/artists/active`, `/api/studios/active`; `seed.ts:4,224-242`). **The absence of Review/AggregateRating markup today is CORRECT and must be preserved until real review data exists. Fake review markup must never be added; seed ratings must never be marked up** — doing so violates Google's review-snippet policies and risks manual actions (R1 F5).
4. **Unverified values are removed, not published.** Per R8 Rec 1: if a business fact (phone, legalName, hours, social URL) is not confirmed, delete the field from schema rather than ship a placeholder.

### 1B. Inventory notes

- **Delivery split:** exactly ONE schema block is render-independent (the static `TattooParlor` in `index.html`). Everything else is injected client-side by `useSEO.ts:69-84` (`document.createElement('script')`, `data-seo="useSEO"`, cleanup on unmount) and therefore invisible to curl, `kimi_fetch_v2`, and FetchURL (R2 UNVERIFIED: FetchURL errored "page may require JavaScript to render").
- **Identity chain:** the static JSON-LD, `business.ts:7-25`, the footer, and the `wa.me` links form one NAP chain — currently poisoned at the source with placeholder data (R1 F6; R8 Table C). Fixing `business.ts` with verified values and re-syncing is the unblocking task for all LocalBusiness/Service schema.
- **Host alignment:** every schema `url`/canonical/OG URL must be rebuilt on the single chosen host after the www/non-www decision (R1 F2; R2 F1; R8 Rec 7).
